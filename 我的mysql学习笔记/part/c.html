键值约束

一、主键约束(primary key)

能够唯一确定一张表中的一条记录，也就是我们通过给某个字段增加约束，就可以使得该字段不重复且不为空；

创建表user，并指定id为主键：
	
	mysql> CREATE TABLE user(
    -> id int primary key,
    -> name varchar(20)
    -> );
	Query OK, 0 rows affected (2.03 sec)
	
插入数据：
	mysql> insert into user values(1,'张三');
	Query OK, 1 row affected (0.49 sec)
	
此时由于主键约束，再插入id相同的值就会失败：	
	mysql> insert into user values(1,'李四');
	ERROR 1062 (23000): Duplicate entry '1' for key 'PRIMARY'

联合主键

创建表user2，并将id和name都设为主键（联合主键）：

	CREATE TABLE user2(
	id int,
	name varchar(20),
	password varchar(20),
	primary key(id,name)
	);
	insert into user2 values(1,'张三','123');
	
	insert into user2 values(2,'张三','123'); --此条也可增加，联合主键只需一条满足即可相当于‘或’但是只要有NULL存在就无法创建。

自增约束(auto_increment)

	CREATE TABLE user3(
		id int primary key auto_increment,
		name varchar(20)
	);
	insert into user3(name)values('张三');
	select * from user3; --可见id自动添加；
	+------+-----+
	| id | name  |
	+------+-----+
	| 1  | 张三  |
	+------+-----+

建表后增加与删除主键

	CREATE TABLE user4(
		id int,
		name varchar(20)
	);
	增加：
		alter table user4 add primary key(id);
		或者
		alter table modify id int primary key;
	使用describe查看表中id的key列：
		describe user4;
		+-------+-------------+------+-----+---------+-------+
		| Field | Type        | Null | Key | Default | Extra |
		+-------+-------------+------+-----+---------+-------+
		| id    | int(11)     | NO   | PRI | NULL    |       |
		| name  | varchar(20) | YES  |     | NULL    |       |
		+-------+-------------+------+-----+---------+-------+
	删除：
	alter table user4 drop primary key;
	describe user4;
	+-------+-------------+------+-----+---------+-------+
	| Field | Type        | Null | Key | Default | Extra |
	+-------+-------------+------+-----+---------+-------+
	| id    | int(11)     | NO   |     | NULL    |       |
	| name  | varchar(20) | YES  |     | NULL    |       |
	+-------+-------------+------+-----+---------+-------+

二、唯一约束（unique）

约束修饰字段的值不可以重复。(使用语法与主键类似)

	CREATE TABLE user5(
	id int,
	name varchar(20),
	unique(name)
	);
	
	删除：
	alter table user5 drop index name;

三、非空约束（not null）

修饰字段不能为空。

四、默认约束（default）

当我们插入字段的时候，如果没有传值，就会使用默认值。

	CREATE TABLE user6(
	id int,
	name varchar(20),
	age int default 10
	);
	insert into user6(id,name) values(1,'张三')；
	select *from user6;                        
		+------+--------+------+
		| id   | name   | age  |
		+------+--------+------+
		|    1 | 张三   |   10 |
		+------+--------+------+

外键约束foreign key...references...）

涉及到两个表：父表，子表（主表，副表）

	CREATE TABLE classes(
	id int primary key,
	name varchar(20)
	);
	CREATE TABLE students(
		id int primary key,
		name varchar(20),
		class_id int,
		foreign key(class_id) references classes(id)
	); 
	insert into classes values(1,'一班');
	insert into classes values(2,'二班');
	insert into classes values(3,'三班');
	
	insert into students values(3001,'张三',3);
	insert into students values(2002,'李四',2);
	insert into students values(1001,'小红',1);

使用内连接inner join查看表格：
	select * from students inner join classes on students.class_id=classes.id;
		+------+--------+----------+----+--------+
		| id   | name   | class_id | id | name   |
		+------+--------+----------+----+--------+
		| 1001 | 小红   |        1 |  1 | 一班   |
		| 2002 | 李四   |        2 |  2 | 二班   |
		| 3001 | 张三   |        3 |  3 | 三班   |
		+------+--------+----------+----+--------+
		3 rows in set (0.00 sec)
